# 阶段2：用户任务/主题分析

> 运行频率：每次创作时运行
> 输入：用户的创作任务或主题描述 + SESSION_PATH
> 输出：结构化的任务洞察JSON

## 路径约定

- **会话路径**: `$SESSION_PATH`（由主协调子代理传递）
- **输出路径**: `$SESSION_PATH/task.json`

## 执行前初始化

确保会话目录存在（由主协调子代理创建）。

## 分析原则

- 不要急于给出答案，先彻底理解问题
- 考虑受众视角，他们为什么要看这个内容？
- 识别差异化机会，如何避免同质化？
- 标注不确定的地方，供后续确认

## 分析维度

### 1. 任务拆解
- **核心诉求**: 用户真正想表达/传递什么？
- **表层需求**: 字面上在说什么
- **深层需求**: 潜在意图是什么
- **内容边界**: 应该包含什么，不应该包含什么

### 2. 6类关键词提取（非常重要，供阶段3使用）

每类3-5个，要求具体、有画面感、能引发共鸣：

| 类别 | 说明 | 好的例子 | 差的例子 |
|-----|------|---------|---------|
| **主题词** | 直接与主题相关的核心词 | "面试紧张" | "紧张" |
| **情感词** | 期望唤起的情感 | "委屈不甘" | "难过" |
| **场景词** | 相关的场景/情境 | "深夜加班" | "工作忙" |
| **人群词** | 涉及的人群标签 | "35岁职场人" | "打工人" |
| **痛点词** | 相关的痛点/困扰 | "干活多没功劳" | "不公平" |
| **爽点词** | 期望的收获/满足感 | "被领导看见" | "成功" |

### 3. 切入角度建议
提供3个不同风险等级的切入角度：
- **安全角度**: 常规但稳妥的切入，适合账号起步期、品牌合作
- **有趣角度**: 有新意但不越界，适合日常创作、追热点
- **大胆角度**: 有争议性但可能出圈，适合寻求突破、引发讨论

## 任务描述技巧


| 描述方式 | 效果 | 示例 |
|---------|------|------|
| 主题式 | 基础，覆盖面广 | "讲讲职场沟通" |
| 观点式 | 方向明确 | "我想说职场中少说话不一定是坏事" |
| 问题式 | 受众导向 | "职场新人不敢说话怎么办" |
| 场景式 | 具体生动 | "开会时领导问你意见你不知道说什么" |
| 情绪式 | 共鸣性强 | "职场里被忽视的憋屈感" |

```json
  "topic": "用户输入的主题",

  "keywords": {

    "audience": ["人群词1", "人群词2"],

```

## 后续衔接

输出的JSON将作为阶段3"联想重组"的核心输入。确保：
1. 关键词覆盖6个维度
2. 每个关键词都足够具体
3. JSON格式正确可解析
